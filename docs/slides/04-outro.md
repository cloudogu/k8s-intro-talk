## Key takeaways

* **k8s** - run containers in a cluster (on multiple nodes)  
* **pod** - smallest resource in k8s (comprising containers)
* **deployment**
  * scaling out containers
  * rolling updates
  * self-healing
* **service**
  * cluster access
  * load balancing
